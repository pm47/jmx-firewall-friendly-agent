Idea taken from:

    "Connecting Through Firewall Using JMX - Without modifying the server application":http://blogs.sun.com/jmxetc/entry/connecting_through_firewall_using_jmx

to use agent execute:

<pre><code>
    java -Djmx.host=<DESIRED_HOSTNAME> -Djava.rmi.server.hostname=<DESIRED_HOSTNAME> -Djmx.port=<DESIRED_PORT> -javaagent:<PATH_TO_AGENT_JAR> ... (other stuff)
</code></pre>	

Example (tunneling friendly):

<pre><code>
    java -Djmx.port=22000 -Djava.rmi.server.hostname=localhost -javaagent:jmx-firewall-friendly-agent-1.0-SNAPSHOT.jar ... (other stuff)
</code></pre>	

You should use this URL then (after tunneling chosen port):

<pre><code>
    service:jmx:rmi://localhost:22000/jndi/rmi://localhost:22000/jmxrmi
</code></pre>
